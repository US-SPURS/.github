# Getting Support

Thank you for using US-SPURS open source projects! This document explains how to get help with our projects.

## How to Get Help

### Documentation

First, please check the documentation:

* **Project README**: Each repository has a README.md with project-specific information
* **Wiki**: Some projects may have additional documentation in the repository wiki
* **Issues**: Check existing issues - your question may already be answered

### Ask Questions

If you can't find the answer in the documentation:

1. **Check Existing Issues**: Search the repository's issues to see if someone has already asked your question
2. **Open a New Issue**: If you don't find an answer, open a new issue using the "Question" template
3. **Be Specific**: Include as much detail as possible about what you're trying to do and what you've already tried

### Report Bugs

If you've found a bug:

1. **Check Existing Issues**: Make sure it hasn't already been reported
2. **Use the Bug Report Template**: This helps us gather the information we need
3. **Include Details**: Operating system, version numbers, steps to reproduce, etc.
4. **Be Patient**: Our maintainers are often working on multiple projects

### Request Features

Have an idea for a new feature?

1. **Check Existing Issues**: Someone may have already requested it
2. **Use the Feature Request Template**: This helps us understand your needs
3. **Explain the Use Case**: Help us understand why this feature would be valuable
4. **Be Open to Discussion**: We may suggest alternative approaches

## Contact Information

### Email Support

* **General Questions**: opensource@us-spurs.gov
* **Security Issues**: security@us-spurs.gov (for vulnerabilities only - see [SECURITY.md](SECURITY.md))
* **Press Inquiries**: press@us-spurs.gov

### Response Time

We strive to respond to all inquiries within:

* **Security Issues**: 48 hours
* **Bug Reports**: 5 business days
* **Feature Requests**: 10 business days
* **General Questions**: 10 business days

Please note that these are goals, not guarantees. Response times may vary based on complexity and current workload.

## Community Guidelines

When seeking support:

* **Be Respectful**: Follow our [Code of Conduct](CODE_OF_CONDUCT.md)
* **Be Patient**: Maintainers and community members are volunteers or have other responsibilities
* **Be Clear**: Provide enough context and detail for others to understand your question
* **Be Helpful**: If you solve your own problem, share the solution to help others
* **Give Back**: Consider helping others once you're more familiar with the project

## What We Can't Help With

Please note that:

* We cannot provide support for modified versions of our software
* We cannot provide security support for deprecated versions
* We cannot provide general IT support unrelated to our specific projects
* We cannot guarantee support for third-party integrations or modifications
